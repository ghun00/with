# 보고서 모달 UI 및 Markdown 에디터 수정안

현재 구현된 보고서 모달을 레퍼런스(img_reportRefer.png)와 같은 문서 중심 UI로 개선해주세요.

- 현재 구현된 디자인은 최대한 유지하며 수정이 필요한 부분(타이틀 위치 수정, markdown 기반 에디터 적용)만 수정해주세요.

레퍼런스의 전체 기능을 그대로 추가하는 것이 아니라, **상단 툴바와 문서 본문이 분리된 레이아웃 및 문서 편집 경험**을 중심으로 참고해주세요.

## 1. 모달 상단 바

현재 상단 바 왼쪽에 노출되는 보고서 제목을 본문 영역으로 이동합니다.

상단 바에는 문서 조작 기능만 배치합니다.

### 열람 상태

- 복사 아이콘
- PDF 다운로드 아이콘
- 편집 버튼
- 닫기 아이콘

### 편집 상태

- 저장 버튼
- 취소 버튼
- 닫기 아이콘

### 적용 기준

- 기능 버튼은 상단 우측에 정렬
- 상단 바는 본문 스크롤 시에도 고정
- 각 아이콘에 `복사`, `PDF 다운로드`, `닫기` 툴팁 제공
- 레퍼런스에 있는 보기 위치 변경, 이전·다음 이동 등의 기능은 현재 범위에서 제외

## 2. 보고서 제목 및 기본정보 배치

현재 `한지훈 · 천안쌍용고 3학년`이 표시되는 본문 최상단 위치에 보고서 제목을 배치합니다.

```text
2026.07.19 상담보고서

한지훈 · 천안쌍용고 3학년
상담 일시 2026.07.19 | 담당 컨설턴트 한지훈 | 직접 작성
```

### 적용 기준

- 보고서 제목을 본문 최상위 `H1` 스타일로 표시
- 학생 이름·학교·학년은 제목 하단의 보조 정보로 이동
- 상담 일시, 담당 컨설턴트, 작성 방식은 다음 행에 표시
- 기본정보 하단에 구분선 적용
- 제목과 기본정보를 실제 보고서 콘텐츠의 시작 영역으로 구성
- 편집 상태에서는 보고서 제목을 직접 수정할 수 있도록 적용

## 3. Markdown 기반 보고서 에디터

보고서 본문은 Markdown 기반으로 저장하고 렌더링되도록 변경합니다.

단, 사용자가 Markdown 원문만 보는 코드 에디터 형태가 아니라, **노션처럼 Markdown 문법 입력 시 즉시 문서 서식으로 변환되는 WYSIWYG 에디터**로 구성합니다.

### 지원 형식

- `# ` → 제목 1
- `## ` → 제목 2
- `### ` → 제목 3
- `- ` 또는 `* ` → 글머리 기호 목록
- `1. ` → 번호 목록
- `[] ` → 미완료 체크리스트
- `> ` → 인용문
- `---` → 구분선
- `**텍스트**` → 굵게
- `*텍스트*` → 기울임

예를 들어 사용자가 빈 줄에서 `- `를 입력하면 즉시 글머리 기호 목록으로 전환하고, `- [ ] `를 입력하면 체크박스가 있는 To Do 항목으로 전환합니다.

## 4. Markdown 편집 동작

- Markdown 문법 입력 후 `Space` 입력 시 해당 블록 형식으로 자동 변환
- 목록에서 `Enter` 입력 시 같은 형식의 다음 항목 생성
- 내용이 없는 목록에서 `Enter`를 다시 입력하면 일반 본문으로 전환
- 체크박스 클릭 시 완료·미완료 상태 변경
- `###`로 생성된 소제목도 일반 텍스트처럼 수정·삭제 가능
- 붙여넣은 Markdown 콘텐츠도 동일한 문서 형식으로 변환
- 열람 상태에서는 Markdown 문법 기호를 노출하지 않고 렌더링된 결과만 표시
- 편집 상태에서도 문법 기호보다 변환된 WYSIWYG 결과를 우선 표시

## 5. 기본 보고서 템플릿

신규 상담보고서 작성 시 기존 항목을 Markdown 템플릿(현재 적용된 것 그대로)으로 적용합니다.

```markdown
### 상담 목적

이번 상담의 목적을 입력해주세요.

### 주요 논의

- 주요 논의 내용을 입력해주세요.

### 학생 현황

현재 활동 및 준비 상황을 입력해주세요.

### 결정 사항

- 결정된 내용을 입력해주세요.

### 학생 To Do

- [ ] 학생이 수행할 활동을 입력해주세요.

### 컨설턴트 To Do

- [ ] 컨설턴트가 수행할 활동을 입력해주세요.

### 다음 상담 계획

다음 상담 일정과 확인할 내용을 입력해주세요.
```

### 적용 기준

- 안내 문구는 Placeholder로 사용하고 실제 저장 데이터에는 포함하지 않음
- 기본 소제목과 블록은 생성 이후 자유롭게 수정·삭제·이동 가능
- AI 상담보고서 결과도 동일한 Markdown 구조로 생성하여 같은 에디터에 표시

## 6. 데이터 저장 및 열람

- 보고서 제목과 기본정보는 각각 필요한 데이터로 관리
- 보고서 본문은 Markdown 구조가 유지되는 형태로 저장
- 저장된 Markdown을 열람할 때 HTML 문서 형태로 렌더링
- 소제목, 목록, 체크리스트, 굵은 글씨 및 구분선 등의 서식 유지
- 기존에 저장된 보고서도 가능한 범위에서 Markdown 형식으로 변환하여 정상적으로 열람할 수 있도록 처리

## 7. 복사 기능

복사 아이콘 클릭 시 다음 내용을 클립보드에 복사합니다.

- 보고서 제목
- 학생 및 상담 기본정보
- 보고서 전체 본문

가능하면 서식 있는 텍스트와 Markdown 원문을 함께 처리하고, 지원되지 않는 환경에서는 일반 텍스트로 복사합니다.

## 8. PDF 다운로드

다운로드 아이콘 클릭 시 Markdown 파일이 아닌 PDF 파일을 생성합니다.

### 처리 방식

- 저장된 Markdown을 화면과 동일한 문서 스타일로 렌더링
- 렌더링된 보고서 콘텐츠를 A4 규격 PDF로 변환
- Markdown 원문 기호가 아니라 실제 적용된 문서 서식을 출력

### PDF 포함 항목

- 보고서 제목
- 학생 이름·학교·학년
- 상담 일시
- 담당 컨설턴트
- 작성 방식
- 보고서 전체 본문

### PDF 제외 항목

- 모달 상단 바
- 복사·다운로드·편집·닫기 버튼
- 에디터 툴바
- Placeholder
- 블록 조작 기능

### 출력 기준

- 파일명: `{상담일자}_{학생명}_상담보고서.pdf`
- A4 기준 여백 적용
- 한글 폰트 정상 출력
- 제목, 소제목, 본문, 목록 및 체크리스트 서식 유지
- 여러 페이지 출력 시 자연스럽게 페이지 분할
- 소제목만 페이지 하단에 남지 않도록 페이지 나눔 처리
- 체크리스트의 완료·미완료 상태가 PDF에도 구분되도록 적용

## 9. 최종 사용자 흐름

### 신규 작성

`상담보고서 작성 → Markdown 템플릿이 적용된 에디터 실행 → 문서 작성 → 저장`

### 저장된 보고서 열람

`보고서 선택 → 렌더링된 문서 열람 → 복사 또는 PDF 다운로드`

### 기존 보고서 수정

`보고서 열람 → 편집 → Markdown 기반 WYSIWYG 에디터 활성화 → 수정 → 저장`

## 핵심 적용 방향

보고서를 여러 입력 필드의 집합이 아닌 **하나의 Markdown 문서로 관리**해주세요.

내부적으로는 Markdown 구조를 유지하되, 사용자에게는 Markdown 원문 편집기가 아닌 **노션처럼 입력 즉시 문서 형식이 적용되는 자연스러운 편집 환경**을 제공해야 합니다.
