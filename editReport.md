상담보고서 UI 3차 수정안

현재 상담보고서 UI를 컨설턴트가 쉽게 작성하고, 저장 이후에는 관리자가 보고서 형태로 편하게 열람할 수 있도록 수정해주세요.

1. 문서 레이아웃
각 항목을 Box로 구분하지 않고 하나의 연속된 문서로 구성
제목, 소제목, 본문이 자연스럽게 이어지는 Rich Text Editor 형태 유지
빈 섹션에 적용된 과도한 최소 높이 제거
내용이 입력됨에 따라 본문 높이가 자동으로 확장되도록 적용
소제목과 다음 소제목 사이에만 일정한 여백 적용
텍스트 계층
보고서 제목: 24~28px / Bold
섹션 소제목: 18~20px / Semi Bold
본문: 15~16px / Regular
본문 줄간격: 1.6~1.8
소제목 위쪽 여백은 크게, 본문과의 간격은 비교적 작게 적용
2. 보고서 기본정보

보고서 제목 아래에 다음 기본정보를 간결하게 노출해주세요.

대상 학생: 학생 이름, 학교, 학년
상담 일시
담당 컨설턴트
작성 방식: 직접 작성 / AI 생성

기존 상담 일시 본문 섹션은 상단 기본정보 영역으로 이동합니다.

기본정보는 별도 카드 형태보다 문서의 메타데이터처럼 간결하게 구성해주세요.

상담 일시는 수정이 가능해야하며 수정 시, 캘린더 형태의 picker를 선택할 수 있도록 합니다.

3. 기본 템플릿 및 Placeholder

기존 상담보고서 항목을 H3 소제목으로 적용하고, 각 항목 아래에 입력 목적에 맞는 placeholder를 제공합니다.

상담 목적: 이번 상담의 목적을 입력해주세요.
주요 논의: 학생과 논의한 주요 내용을 입력해주세요.
학생 현황: 현재 활동 및 준비 상황을 입력해주세요.
결정 사항: 상담을 통해 결정된 내용을 입력해주세요.
학생 To Do: 학생이 수행해야 할 항목을 입력해주세요.
컨설턴트 To Do: 컨설턴트가 수행해야 할 항목을 입력해주세요.
다음 상담 계획: 다음 상담 일정과 확인할 내용을 입력해주세요.

Placeholder는 실제 저장 데이터에 포함하지 않으며, 내용 입력 시 자동으로 사라지도록 적용합니다.

4. 섹션별 기본 형식

섹션의 내용에 맞는 기본 입력 형식을 제공합니다.

상담 목적: 일반 문단
주요 논의: 글머리 기호 목록
학생 현황: 일반 문단
결정 사항: 글머리 기호 목록
학생 To Do: 체크리스트
컨설턴트 To Do: 체크리스트
다음 상담 계획: 일반 문단

기본 형식은 사용자가 편집 과정에서 자유롭게 변경할 수 있어야 합니다.

5. 블록 편집 UX
블록 조작 기능은 항상 노출하지 않음
해당 블록에 마우스를 올리거나 선택한 경우에만 조작 기능 노출
블록 왼쪽에 ⋮⋮ 형태의 핸들 제공
핸들을 통해 다음 기능 제공
본문·소제목 등 형식 변경
위·아래 순서 이동
블록 삭제
일반 열람 상태에서는 블록 핸들과 편집 기능을 노출하지 않음
6. 최초 작성 상태

저장되지 않은 신규 상담보고서는 작성 상태로 실행합니다.

상단 버튼
저장
닫기

신규 보고서는 아직 복사하거나 다운로드할 데이터가 확정되지 않은 상태이므로 다음 버튼을 노출하지 않습니다.

복사
다운로드
편집

저장 버튼 클릭 시 보고서를 생성하고 상담보고서 목록에 반영합니다.

내용을 변경한 상태에서 닫기를 누르면 저장되지 않은 변경 사항이 있다는 확인 안내를 제공합니다.

7. 저장된 보고서 열람 상태

저장이 완료된 보고서를 열면 기본적으로 열람 상태로 실행합니다.

상단 버튼
복사 아이콘
다운로드 아이콘
편집 버튼
닫기 아이콘

복사와 다운로드는 텍스트 버튼이 아닌 아이콘 버튼으로 적용하고, 마우스를 올렸을 때 각각 복사, 다운로드 툴팁을 노출합니다.

기능
복사: 보고서 제목, 기본정보 및 전체 본문을 클립보드에 복사
다운로드: 저장된 보고서를 파일로 다운로드
편집: 열람 상태를 편집 상태로 전환
8. 저장된 보고서 편집 상태

편집 버튼 클릭 시 동일한 문서 화면에서 내용을 수정할 수 있도록 전환합니다.

상단 버튼
저장
취소
닫기 아이콘

편집 중에는 복사와 다운로드 아이콘을 숨깁니다.

저장: 수정 내용을 반영하고 열람 상태로 전환
취소: 변경 내용을 반영하지 않고 기존 열람 상태로 복귀
닫기: 저장되지 않은 변경 사항이 있다면 확인 안내 노출
9. AI 상담보고서
AI 생성 결과도 동일한 문서 레이아웃과 템플릿 구조로 구성
AI 생성 완료 직후에는 저장 전 편집 상태로 노출
사용자가 생성 결과를 검토하고 수정한 뒤 저장하도록 적용
최초 저장 전에는 복사 및 다운로드 아이콘 미노출
저장 완료 후 일반 보고서와 동일한 열람·편집·복사·다운로드 기능 제공
상태별 상단 기능 정리
상태	노출 기능
신규 직접 작성	저장, 닫기
AI 초안 생성 직후	저장, 닫기
저장된 보고서 열람	복사 아이콘, 다운로드 아이콘, 편집, 닫기
저장된 보고서 편집	저장, 취소, 닫기

---
"상담·월간 보고서 UI 수정안"

현재 상담보고서 UI를 GPT Canvas와 유사한 문서 열람·편집 환경으로 개선하고, 월간 보고서에도 동일한 UI를 적용해주세요.

1. 보고서 상단 헤더

현재 분리된 상단 툴바와 본문 제목 영역을 하나의 헤더로 통합합니다.

2026.07.19 상담보고서          복사  다운로드  편집  닫기
보고서 제목을 헤더 왼쪽에 배치
복사, 다운로드, 편집, 닫기 기능을 헤더 오른쪽에 배치
제목과 기능 버튼이 동일한 높이의 한 행에 위치하도록 정렬
본문 상단에 중복으로 노출되는 보고서 제목 제거
헤더는 본문 스크롤 시에도 모달 상단에 고정
현재 헤더에 존재하는 불필요한 빈 공간 제거
2. 헤더 기능
열람 상태

헤더 오른쪽에 다음 기능을 노출합니다.

복사 아이콘
PDF 다운로드 아이콘
편집 버튼
닫기 아이콘

복사와 다운로드는 텍스트가 아닌 아이콘 버튼으로 적용하고, 마우스를 올렸을 때 각각 복사, PDF 다운로드 툴팁을 표시합니다.

편집 상태

편집 상태에서는 복사 및 다운로드 아이콘을 숨기고 다음 기능을 제공합니다.

저장
취소

저장하지 않은 변경 사항이 있는 경우 헤더에 저장되지 않은 변경 사항 상태를 표시합니다.

3. 보고서 기본정보

보고서 제목이 헤더로 이동하므로 본문은 기본정보 영역부터 시작합니다.

현재 여러 행으로 구분된 기본정보를 최대 두 줄 이내로 간결하게 구성해주세요.

상담보고서 예시
한지훈 · 천안쌍용고 3학년
상담 일시 2026.07.19  |  담당 컨설턴트 한지훈  |  직접 작성
월간 보고서 예시
한지훈 · 천안쌍용고 3학년

라벨은 연한 색상으로 처리
실제 데이터는 라벨보다 명확하게 표시
기본정보 하단에 구분선 적용
본문 내용을 읽는 데 방해되지 않도록 간결하게 구성
4. 보고서 모달 레이아웃

일반적인 팝업보다 문서 작업에 집중할 수 있는 Canvas형 작업 공간으로 구성합니다.

데스크톱에서 화면 높이의 약 90~95% 사용
헤더는 모달 상단에 고정
헤더를 제외한 본문 영역에만 스크롤 적용
본문 콘텐츠의 최대 폭을 제한하여 긴 문장의 가독성 확보
모바일에서는 전체 화면 모달로 전환
기존 문서형 Rich Text Editor 구조 유지
열람 상태에서는 편집 도구와 블록 조작 기능 숨김
5. PDF 다운로드

다운로드 아이콘 클릭 시 Markdown 파일이 아닌 PDF 파일로 저장되도록 변경합니다.

파일명
{상담일자}_{학생명}_상담보고서.pdf

예시:

2026.07.19_한지훈_상담보고서.pdf

월간 보고서는 다음 형식으로 적용합니다.

{대상연월}_{학생명}_월간보고서.pdf

현재 독립 페이지 형태로 구성된 월간 보고서를 상담보고서와 동일한 모달형 Report UI로 변경합니다.

7. 보고서 유형별 구성
구분	상담보고서	월간 보고서
제목	{상담일자} 상담보고서	{대상연월} 월간보고서

상담보고서와 월간 보고서는 별개의 UI로 구현하지 않고, 가능한 경우 공통 Report Modal 구조를 사용하고 보고서 유형별 제목, 기본정보 및 본문 템플릿만 다르게 적용해주세요.

상태	노출 기능
